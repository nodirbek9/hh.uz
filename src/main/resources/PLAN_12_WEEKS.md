| Разрешено | Запрещено |

**Формат работы с ИИ:**

1. Ты читаешь тему сам (документация, статьи, книги)

3. Ты объясняешь мне в этот чат что сделал

### НЕДЕЛЯ 3 (30 июня — 6 июля): JAVA CORE ГЛУБОКО

```
40+ часов Java Core
```

**Ежедневная структура:**
- 2 часа — чтение темы + конспект СВОИМИ СЛОВАМИ
- 3 часа — практика/задачи
- 2 часа — объяснение темы ВСЛУХ (как будто на собеседовании, запись на диктофон)

**Темы по дням:**

- **День 1: Коллекции — List**
  - ArrayList vs LinkedList: внутреннее устройство, сложность операций (get, add, remove), когда что использовать
  - Vector, Stack (legacy, почему не используют)
  - Практика: реализовать свой ArrayList с нуля

- **День 2: Коллекции — Map и Set**
  - HashMap: хэширование, бакеты, коллизии, красно-чёрное дерево (когда >= 8 элементов в бакете), load factor (0.75), resize (удвоение), capacity
  - HashSet — почему внутри HashMap
  - TreeMap/TreeSet — красно-чёрное дерево, сложность O(log n)
  - Контракт equals() и hashCode(): почему если переопределил equals, ОБЯЗАН переопределить hashCode
  - Практика: задачи на HashMap, объяснить контракт

- **День 3: Иерархия коллекций + GC + Память**
  - Иерархия: Iterable → Collection → List/Set/Queue. Map отдельно.
  - Heap vs Stack: что где хранится (объекты в heap, ссылки и примитивы в stack)
  - GC: Eden → Survivor S0/S1 → Old Generation → Metaspace
  - Serial GC, Parallel GC, G1 GC, ZGC — когда какой, что используется по умолчанию в Java 17/21 (G1)
  - String Pool: где находится (heap с Java 7), как работает интернирование строк

- **День 4: Stream API + Функциональные интерфейсы**
  - Промежуточные vs терминальные операции
  - map, filter, reduce, collect, flatMap, peek, forEach, sorted, distinct, limit, skip
  - Ленивость стримов (метод НЕ выполняется до терминальной операции)
  - Функциональные интерфейсы: Predicate<T>, Function<T,R>, Consumer<T>, Supplier<T>, UnaryOperator<T>, BinaryOperator<T>
  - Практика: 15 задач на Stream API

- **День 5: Многопоточка**
  - Thread, Runnable, Callable (разница)
  - volatile — гарантия видимости (НЕ атомарности)
  - synchronized — блокировка на уровне объекта/класса
  - Deadlock: что это, как происходит, как предотвратить
  - Race Condition: пример с counter++
  - ExecutorService: FixedThreadPool, CachedThreadPool, ScheduledThreadPool, SingleThreadExecutor
  - CompletableFuture — базово (runAsync, supplyAsync, thenApply, thenAccept)

- **День 6: ООП + SOLID + Исключения + Дженерики**
  - 4 принципа ООП с примерами из твоих проектов:
    - Инкапсуляция (private поля + getters/setters)
    - Наследование (extends)
    - Полиморфизм (overriding, overloading, один интерфейс — много реализаций)
    - Абстракция (абстрактные классы, интерфейсы)
  - SOLID: каждый принцип с примером
    - S: Один класс — одна ответственность
    - O: Открыт для расширения, закрыт для модификации
    - L: Принцип подстановки Барбары Лисков
    - I: Разделение интерфейсов (много маленьких лучше одного большого)
    - D: Зависимость от абстракций, не от реализаций
  - Checked vs Unchecked exceptions (IOException vs NullPointerException)
  - try-with-resources (AutoCloseable)
  - Дженерики: `<T>`, wildcards (`? extends`, `? super`), type erasure, PECS (Producer Extends, Consumer Super)

- **День 7: Повторение + LeetCode + Запись ответов**
  - 15 задач LeetCode (Easy + Medium)
  - Запись ответов на диктофон по всем темам недели
  - Прослушать, отметить что звучит неуверенно → переделать

### НЕДЕЛЯ 5 (14–20 июля): SPRING CORE + SPRING BOOT ВНУТРИ

```
40+ часов Spring Internals
```

- **День 1: IoC и DI**
  - Что такое Inversion of Control, контейнер
  - BeanFactory vs ApplicationContext
  - Способы внедрения: Constructor, Setter, Field
  - Почему Constructor Injection лучше (тестирование, иммутабельность, циклические зависимости)
  - @Autowired, @Qualifier, @Primary

- **День 2: Жизненный цикл бина**
  - Полный цикл: BeanDefinition → BeanFactoryPostProcessor → BeanPostProcessor (postProcessBeforeInitialization) → @PostConstruct → InitializingBean → BeanPostProcessor (postProcessAfterInitialization) → Ready → @PreDestroy → DisposableBean
  - Scope: Singleton (по умолчанию), Prototype, Request, Session, Application
  - Проблема: внедрение Prototype в Singleton (proxyMode = ScopedProxyMode.TARGET_CLASS)
  - BeanPostProcessor, BeanFactoryPostProcessor — зачем нужны, примеры использования

- **День 3: @Transactional ГЛУБОКО**
  - Как работает: Spring создаёт прокси (JDK Dynamic Proxy или CGLIB)
  - Propagation: REQUIRED (по умолчанию), REQUIRES_NEW, NESTED, MANDATORY, NEVER, NOT_SUPPORTED, SUPPORTS
  - Isolation: DEFAULT, READ_UNCOMMITTED, READ_COMMITTED, REPEATABLE_READ, SERIALIZABLE
  - Read-only транзакции (оптимизация Hibernate: dirty checking отключается)
  - Почему @Transactional не работает при вызове метода из того же класса (self-invocation, прокси не перехватывает)
  - Почему HTTP-запрос внутри транзакции — плохо (держит соединение с БД, блокировки)

- **День 4: Spring Boot Autoconfiguration**
  - @SpringBootApplication = @Configuration + @EnableAutoConfiguration + @ComponentScan
  - Как работает автоконфигурация: spring.factories → @ConditionalOnClass, @ConditionalOnMissingBean
  - Анализ стартера (spring-boot-starter-data-jpa, spring-boot-starter-web)
  - application.yml vs application.properties. Профили (application-dev.yml)
  - Actuator: health, metrics, info endpoints

- **День 5: Spring Data JPA внутри**
  - Как Hibernate генерирует SQL из имени метода (findByEmailAndAge → парсинг → JPQL → SQL)
  - JPQL vs Native Query: @Query, @Modifying
  - Прокси-объекты Hibernate, ленивая загрузка (FetchType.LAZY → proxy, FetchType.EAGER → join)
  - LazyInitializationException: почему возникает (сессия закрылась), как решить (@Transactional, JOIN FETCH, EntityGraph, OSIV)
  - Спецификации (JpaSpecificationExecutor): Criteria API, Predicate, динамическая фильтрация

- **День 6: Циклические зависимости + AOP + Spring Security базово**
  - Циклическая зависимость бинов: почему возникает (A → B → A), как решить (@Lazy, Setter injection, рефакторинг)
  - AOP: Advice (Before, After, Around), Pointcut (execution, annotation), JoinPoint, Aspect, Weaving
  - Spring Security: SecurityFilterChain, цепочка фильтров, SecurityContextHolder, Authentication vs Authorization

### НЕДЕЛЯ 6 (21–27 июля): SPRING SECURITY + POSTGRESQL ГЛУБОКО

```
40+ часов
```

**Spring Security — продолжение (20 часов):**
- JWT глубоко:
  - Структура токена (Header.Payload.Signature)
  - Access token vs Refresh token
  - Процесс аутентификации: login → валидация → генерация JWT → возврат клиенту
  - Валидация JWT на каждом запросе (OncePerRequestFilter)
  - Где хранить токен: httpOnly cookie (безопасно) vs localStorage (XSS-уязвимость)
- @PreAuthorize, @PostAuthorize, @Secured — method-level security
- Role-based vs Permission-based (ROLE_ADMIN vs 'transaction:read')
- OAuth2/OIDC базово (Resource Server, Authorization Server, client_credentials)
- CSRF защита, CORS конфигурация
- PasswordEncoder: BCrypt (почему, параметр strength, salt)

**PostgreSQL глубоко (20 часов):**
- Как PostgreSQL работает под капотом:
  - Процесс обработки запроса: Parser → Rewriter → Planner → Executor
  - MVCC (Multiversion Concurrency Control): как PostgreSQL хранит версии строк (xmin, xmax), снепшоты
  - VACUUM, VACUUM FULL, AUTOVACUUM: зачем нужно, что делает, когда запускать
  - Dead tuples, bloat — как таблицы раздуваются
- Типы индексов: B-tree (по умолчанию), Hash, GIN (полнотекстовый поиск), GiST (геоданные), BRIN (большие таблицы)
  - Когда какой использовать
  - Как индекс ускоряет запрос (B-tree → O(log n) вместо Seq Scan O(n))
- EXPLAIN и EXPLAIN ANALYZE:
  - Чтение плана запроса: Seq Scan, Index Scan, Index Only Scan, Bitmap Scan, Nested Loop, Hash Join, Merge Join
  - cost, actual time, rows, buffers
  - Оптимизация запроса на основе EXPLAIN
- Connection Pool (HikariCP):
  - Как работает: пул соединений, переиспользование
  - Параметры: maximumPoolSize, minimumIdle, connectionTimeout, idleTimeout
  - Проблема "connection leak" (забыли закрыть соединение)
